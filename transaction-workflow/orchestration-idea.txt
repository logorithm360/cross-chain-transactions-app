## What This Actually Is

This is not a CRE workflow. CRE workflows compile to WASM and run on the DON — they have no concept of interactive terminal input, no `readline`, no `process.stdin`. You cannot build a press-1-press-2 menu inside a CRE workflow.

What you are building is a **local CLI orchestrator** that:
- Lives outside the CRE workflow layer
- Runs on the user's machine as a Node/Bun process
- Connects to MetaMask or a local wallet
- Reads user input interactively
- Calls the deployed CRE workflows via their HTTP endpoints
- Calls smart contracts directly via RPC for read operations
- Handles the full conversation with the user from start to finish

The relationship between the orchestrator and the CRE workflows:

```
USER
  ↕  (terminal interaction)
CLI ORCHESTRATOR  (runs locally — this is what we are building)
  ↕  (HTTP calls to CRE workflow endpoints)
CRE WORKFLOWS    (deployed on the DON — the four services)
  ↕  (EVM calls)
SMART CONTRACTS  (deployed on Sepolia/Amoy/etc)
```
